1. 基础信息

LoginService
https://aichat-login-itosyagchs.cn-hangzhou.fcapp.run
发送验证码、手机号验证码登录、签发 JWT
ChatService
https://aichat-service-imefiprtzg.cn-hangzhou.fcapp.run
角色列表、聊天、流式聊天、历史记录、清空当前聊天

1.1 当前线上联调状态

  1. LoginService GET /health 返回 200，configured=true。
  2. ChatService GET /health 返回 200，configured=true。
  3. ChatService 当前要求 JWT 鉴权，authRequired=true。
  4. ChatService GET /api/chat/roles 当前返回 4 个有效角色。
  5. ChatService 当前数据库和天气能力均已联通。

2. 鉴权约定

推荐统一用请求头传 JWT：

Authorization: Bearer <token>

ChatService 兼容在 Body 中传 token，但不建议作为正式接入方式。

2.1 不需要 JWT 的接口

  1. LoginService 的全部接口
  2. ChatService GET /
  3. ChatService GET /health
  4. ChatService GET /api/chat/roles

2.2 需要 JWT 的接口

  1. GET /api/chat/history
  2. POST /api/chat/clear
  3. POST /api/chat/completions
  4. POST /api/chat/stream

5.5 POST /api/chat/completions

字段类型必填说明
roleKeystring否角色 Key，默认 chenge
messagestring否当前用户输入文本，优先级最高
messagesarray否可选消息数组，服务端会取最后一个用户文本
modelstring否指定模型
temperaturenumber否温度参数
topPnumber否Top P
maxTokensnumber否最大输出 Token 数
presencePenaltynumber否presence penalty
frequencyPenaltynumber否frequency penalty
weatherCitystring否显式指定天气查询城市
imageUrlstring否图片 URL，只支持单张
imageDataUrlstring否Base64 Data URL，只支持单张，优先级高于 imageUrl
tokenstring否Body 中传 JWT 的兼容字段，不推荐
streamboolean否设为 true 时，行为等价于流式接口
请求规则
1. message 有值时优先使用。
2. 不传 weatherCity 时，只有明显的天气问题才会触发天气工具。
3. 当前只支持单张图片，图片不做持久化。
4. /api/chat/completions 传 stream=true 时，行为等价于流式接口。

5.6 POST /api/chat/stream

请求参数与 POST /api/chat/completions 完全一致，返回 text/event-stream。

前端主要解析 choices[0].delta.content，结束帧固定为 data: [DONE]。

6. 错误码和返回规范

状态码场景说明
400参数错误例如手机号格式不正确、缺少必要参数
401未登录或登录态无效例如缺少 JWT、JWT 无效、验证码校验失败
500服务内部错误或配置缺失例如数据库未配置、内部异常
502上游服务失败例如模型服务或天气服务调用失败
504模型请求超时例如大模型在服务端超时阈值内未返回

7.5 流式聊天

const res = await fetch('https://aichat-service-imefiprtzg.cn-hangzhou.fcapp.run/api/chat/stream', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${token}`
  },
  body: JSON.stringify({
    roleKey: 'dongxuelian_like',
    message: '陪我聊聊天吧。'
  })
});

const reader = res.body.getReader();
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { value, done } = await reader.read();
  if (done) break;

  buffer += decoder.decode(value, { stream: true });
  const frames = buffer.split('\n\n');
  buffer = frames.pop() || '';

  for (const frame of frames) {
    if (!frame.startsWith('data: ')) continue;
    const raw = frame.slice(6);
    if (raw === '[DONE]') continue;

    const event = JSON.parse(raw);
    const delta = event.choices?.[0]?.delta?.content || '';
    if (delta) {
      console.log(delta);
    }
  }
}

8. 前端对接建议

  1. 角色相关素材全部走 GET /api/chat/roles，不要在前端保存静态副本。
  2. 所有受保护接口统一走 Authorization: Bearer <token>。
  3. 收到 401 时，前端应清除本地 JWT 并跳转登录。
  4. 需要天气时，最好显式传 weatherCity，不要完全依赖模型自动判断。
  5. 图片聊天只支持单图，建议上传前压缩，避免请求体过大。
  6. 对 502 和 504 做重试提示，不要直接把上游错误原样暴露给用户。
